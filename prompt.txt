# 新機能：収穫逓減（スケール時の効率悪化・質低下）ロジックの導入
「質の良い媒体でも、一定規模を超えて出稿を広げるとターゲットが広がりすぎてCPAが悪化し、面予率（質）も低下する」という実務上の懸念をシミュレーションに組み込むため、区分線形近似を用いたペナルティロジックを導入してください。

1. 【入力パラメータ（st.data_editor）の追加】
入力データテーブルに以下の3カラムを追加してください。
- `安全拡大件数` (int): この件数までは設定通りのCPA・面予率を維持できるボーダーライン。（デフォルトは `上限件数` と同じ値にする）
- `超過時CPA悪化率(%)` (float): 安全拡大件数を超えた分の件数に対して、CPAが何%割増になるか。（デフォルト例: 120%）
- `超過時面予低下率(%)` (float): 安全拡大件数を超えた分の件数に対して、面予率が何%掛けになるか。（デフォルト例: 80%）

2. 【数理最適化（PuLP）への組み込み（区分線形近似）】
各媒体 $i$ の獲得件数を、以下の2つの変数に分割して定義してください。
- `x_base[i]`: 安全圏内の獲得件数 (0 <= x_base[i] <= 安全拡大件数)
  -> 適用されるCPA = 通常のCPA、面予率 = 通常の面予率
- `x_extra[i]`: 安全圏を超えた獲得件数 (0 <= x_extra[i] <= max(0, 上限件数 - 安全拡大件数))
  -> 適用されるCPA = 通常のCPA * (超過時CPA悪化率 / 100)
  -> 適用される面予率 = 通常の面予率 * (超過時面予低下率 / 100)
- 媒体の総獲得件数 = `x_base[i] + x_extra[i]`
- 媒体の総広告費 = `(通常のCPA * x_base[i]) + (悪化後CPA * x_extra[i])`
※目的関数（粗利最大化等）や制約条件（全体のCPA、目標件数±50件）の計算式は、上記の総獲得件数と総広告費を用いて再構築してください。

3. 【出力結果の集計とUI表示】
- シミュレーション結果の各媒体のテーブル表示において、「目標獲得件数」は `x_base` と `x_extra` の合計値を表示してください。
- また、その媒体の実効CPA（総広告費 ÷ 総獲得件数）と実効面予率（加重平均）を再計算して表示し、安全件数を超えて無理に拡大した媒体の効率が悪化していることがユーザーにわかるようにしてください。
